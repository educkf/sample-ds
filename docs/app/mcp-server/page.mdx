
# MCP Server

AI/LLM integration via Model Context Protocol for seamless Design System development assistance.

## ü§ñ What is MCP (Model Context Protocol)?

The **Model Context Protocol (MCP)** is an open standard for connecting AI assistants and large language models (LLMs) to external data sources and tools. It enables AI assistants like Cursor, Claude, and others to access real-time information and interact with specific services or APIs.

Our MCP Server provides AI assistants with direct access to the Sample Design System documentation, component information, and utility tools, making it easier for developers to get contextual help while building applications.

## üèóÔ∏è How Our MCP Server is Built

Our MCP server is built using modern web technologies and deployed on Vercel:

### Architecture
- **Framework**: Built with the `@vercel/mcp-adapter` for seamless integration
- **Language**: TypeScript with full type safety

- **Deployment**: Vercel Edge Functions for global availability

### Production Endpoint

The server exposes multiple HTTP endpoints (GET, POST, DELETE) to handle different MCP transport protocols and ensure compatibility with various AI assistants.

## üõ†Ô∏è Available Tools

Our MCP server provides **2 powerful tools** for Design System development:

### 1. üì¶ **list_components**
- **Purpose**: Get an overview of all available Design System components
- **Parameters**: None required
- **Returns**: Complete list of 6 components with descriptions, features, and navigation links
- **Includes**: Button, Card, Alert, Accordion, Breadcrumb, Modal components

- **Returns**: 
  - Complete property tables with types and defaults
  - Usage examples and code snippets
  - Installation instructions (NPM + Script tags)
  - Framework compatibility information
  - Accessibility and theming details

## üéØ Adding MCP Server to Cursor

Follow these steps to integrate our MCP server with Cursor AI:

### Step 1: Open Cursor Settings
1. Open Cursor IDE
2. Open File + Preferences + Cursor Settings

### Step 2: Add MCP Server Configuration
Add our server configuration to your MCP settings:

```json
{
  "mcpServers": {
    "sample-design-system-server": {
      "url": "https://sample-ds.vercel.app/mcp",
      "transport": "http"
    }
  }
}
```

### Step 3: Restart Cursor
1. Close and restart Cursor IDE
2. The MCP server will be automatically detected
3. You should see "sample-design-system" in your available tools

### Step 4: Verify Connection
Test the connection by asking Cursor:
- "List all available components in the design system"
- "Show me details about the button component"

## üÜò Troubleshooting

### Common Issues

**MCP Server Not Detected**
- Verify the URL is exactly: `https://sample-ds.vercel.app/mcp`
- Check Cursor's MCP configuration syntax
- Restart Cursor after configuration changes

**Tool Calls Not Working**
- Ensure internet connectivity to reach our Vercel deployment
- Check if the server is responding: visit https://sample-ds.vercel.app
- Verify tool names match exactly: `list_components`, `get_component_details`

**Parameter Validation Errors**
- Component names must be lowercase: `button`, not `Button`
- Check parameter types match the expected schema

### Getting Help
If you encounter issues with our MCP server integration, the server provides detailed error messages to help diagnose problems. All responses include clear formatting and helpful context for troubleshooting.
