
# How to Use Sample Design System

A modern web component library built with Lit Framework that works everywhere - from modern bundlers to simple script tags.

## üéØ Two Ways to Use

Our Design System supports **dual usage patterns** to fit any project:

1. **üì¶ NPM Package** - For modern build tools (Vite, Webpack, Parcel, etc.)
2. **üéØ Script Tags** - For direct HTML usage without build processes

## üì¶ NPM Package Installation

### Install the Package
```bash
npm install sample-design-system-educkf
```

### Usage Options

#### Option 1: Import Individual Components (Recommended)
```javascript
// Import and auto-register specific components
import 'sample-design-system-educkf/components/sample-button';

// Now use in your HTML/templates
// <sample-button variant="primary">Click me!</sample-button>
```

**Benefits:**
- ‚úÖ Optimal bundle size (only what you use)
- ‚úÖ Tree-shaking friendly
- ‚úÖ Automatic component registration

#### Option 2: Import Component Classes
```javascript
// Import component classes for manual control
import { SampleButton } from 'sample-design-system-educkf';

// Register with custom tag name
customElements.define('my-button', SampleButton);

// Or use the class directly
const button = new SampleButton();
document.body.appendChild(button);
```

**Benefits:**
- ‚úÖ Custom element names
- ‚úÖ Programmatic usage
- ‚úÖ Full control over registration

#### Option 3: Import Everything
```javascript
// Import and register all components
import 'sample-design-system-educkf';

// All components are now available
// <sample-button>, <sample-card>, etc.
```

**Benefits:**
- ‚úÖ Simple single import
- ‚úÖ All components available immediately

### Bundle Information
- **Library size**: ~15KB (Lit externalized for optimal bundling)
- **Dependencies**: `lit` (peer dependency)
- **Format**: ES modules with tree-shaking support

### CDN Usage

#### Individual Components
```html

<html>
<body>
  <sample-button variant="primary">Click me!</sample-button>
  
  <!-- Load only the component you need (~21KB) -->

```

#### Complete Library
```html
<!DOCTYPE html>
<html>
<body>
  <sample-button variant="primary">Primary</sample-button>
  <sample-button variant="secondary">Secondary</sample-button>
  
  <!-- Load complete library (~21KB) -->
  <script src="https://unpkg.com/sample-design-system-educkf/dist/standalone/index.js"></script>

</html>
```

### Local Files
```html
<!-- Download and host locally -->
<script src="./js/sample-button.js"></script>
```

### Bundle Information  
- **Individual components**: ~21KB each (self-contained with Lit)
- **Complete library**: ~21KB (all components + Lit bundled)
- **Format**: UMD (works everywhere)

## üöÄ Framework Integration

### React
```jsx
import React from 'react';
import 'sample-design-system-educkf/components/sample-button';

function App() {
  const handleClick = (e) => {
    console.log('Button clicked:', e.detail);
  };

  return (
    <div>
      <sample-button 
        variant="primary" 
        size="large"
        onClick={handleClick}
      >
        React + Web Component
      </sample-button>
    </div>
  );
}

export default App;
```

## üîß Advanced Usage

### TypeScript Integration
```typescript
// types.d.ts
import { SampleButton, SampleCard } from 'sample-design-system-educkf';

declare global {
  namespace JSX {
    interface IntrinsicElements {
      'sample-button': React.DetailedHTMLProps<React.HTMLAttributes<SampleButton>, SampleButton>;
      'sample-card': React.DetailedHTMLProps<React.HTMLAttributes<SampleCard>, SampleCard>;
    }
  }
}
```

### Custom Event Handling
```javascript
// Listen for component events
document.addEventListener('sample-button-click', (event) => {
  console.log('Global button click:', event.detail);
});

// Component-specific listeners
const button = document.querySelector('sample-button');
button.addEventListener('sample-click', (event) => {
  console.log('Specific button click:', event.detail);
});
```

### Dynamic Component Creation
```javascript
// Create components programmatically
import { SampleButton } from 'sample-design-system-educkf';

const button = new SampleButton();
button.variant = 'primary';
button.textContent = 'Dynamic Button';
button.addEventListener('sample-click', handleClick);

document.body.appendChild(button);
```

## üåç Browser Compatibility

### Polyfills
For older browsers that don't support Web Components:

```html
<!-- Load polyfills before components -->
<script src="https://unpkg.com/@webcomponents/webcomponentsjs/webcomponents-loader.js"></script>
<script src="https://unpkg.com/sample-design-system-educkf/dist/standalone/index.js"></script>
```

### Feature Detection
```javascript
// Check for Web Components support
if ('customElements' in window) {
  // Native support
  import('sample-design-system-educkf/components/sample-button');
} else {
  // Load polyfills first
  import('@webcomponents/webcomponentsjs/webcomponents-loader.js')
    .then(() => import('sample-design-system-educkf/components/sample-button'));
}
```

## üìö Best Practices

### Performance Tips
1. **Import only what you need** for optimal bundle size
2. **Use tree-shaking** with modern bundlers
3. **Lazy load components** not immediately visible
4. **Cache CDN resources** for repeat visits

### Accessibility Guidelines
1. **Always provide meaningful labels** for interactive elements
2. **Use semantic HTML structure** within component slots
3. **Test with keyboard navigation** and screen readers
4. **Maintain proper color contrast** when theming

### Maintenance Recommendations
1. **Pin specific versions** in production
2. **Test component updates** in staging environments
3. **Monitor bundle size** impact when updating
4. **Keep design tokens consistent** across projects

## üõ†Ô∏è Troubleshooting

### Common Issues

#### Components not rendering
- Ensure the component script is loaded
- Check for JavaScript errors in console
- Verify proper tag names (kebab-case)

#### Styling not applied
- Check CSS custom property inheritance
- Verify specificity of CSS selectors
- Ensure dark mode variables are defined

#### Events not firing
- Use correct event names (e.g., `sample-click`)
- Check event bubbling settings
- Verify event listener attachment timing
